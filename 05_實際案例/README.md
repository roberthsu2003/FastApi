# 實際案例：物聯網 (IoT) 數據接收與管理系統

本專案展示了如何結合 **FastAPI** 與 **SQLite** 資料庫，建立一個可用於物聯網 (IoT) 或感測器數據上傳、查詢與導出的後端系統。

---

## 🛠️ 專案功能

- **數據接收**：提供 API 接口，供用戶端/感測設備上傳溫度與亮度數據。
- **數據儲存**：將上傳的數據即時寫入本地 SQLite 資料庫 (`data.db`)。
- **數據查詢**：支援依數量查詢最新上傳的 JSON 格式數據。
- **數據匯出**：支援依數量將數據導出為 CSV 檔案下載。

---

## 🗺️ 專案目錄結構

- [01_fastApi_sqlite_iot/](./01_fastApi_sqlite_iot/) - 後端主要程式與設定檔
  - [main.py](./01_fastApi_sqlite_iot/main.py) - FastAPI 主程式
  - `data.db` - 本地 SQLite 資料庫
  - `requirements.txt` - 專案相依套件

## 📖 API 接口使用範例 (API Endpoints & Examples)

啟動伺服器後（預設運作於 `http://127.0.0.1:8000`），您可以使用以下方式進行測試與資料交互：

### 1. 上傳感測器數據 (Upload Sensor Data)
- **路徑**：`/iot`
- **方法**：`POST` (符合 RESTful 規範，使用 Request Body 傳遞 JSON 數據)
- **請求 JSON 格式**：
  ```json
  {
    "time": "20260706 12:35:10",  // 選填，預設為伺服器當前時間
    "light": 120.5,               // 選填，預設為 0.0
    "temperature": 26.4           // 選填，預設為 0.0
  }
  ```

#### 👉 測試範例 (使用 cURL)
- **請求指令**：
  ```bash
  curl -X 'POST' \
    'http://127.0.0.1:8000/iot' \
    -H 'Content-Type: application/json' \
    -d '{
    "light": 120.5,
    "temperature": 26.4
  }'
  ```
- **預期回應**：
  ```json
  {
    "時間": "20260706 12:35:10",
    "光線": 120.5,
    "溫度": 26.4
  }
  ```

---

### 2. 查詢最新數據 (Query JSON Data)
- **路徑**：`/iot_json/{item_count}`
- **方法**：`GET`
- **參數說明**：
  - `item_count` (必填)：欲查詢的最新資料筆數 (例如最新 5 筆)。

#### 👉 測試範例
- **請求連結**：
  ```
  http://127.0.0.1:8000/iot_json/3
  ```
- **預期回應**：
  ```json
  [
    [3, "20260706 12:35:10", 120.5, 26.4],
    [2, "20260706 10:00:00", 98.0, 25.5]
  ]
  ```
  *(回傳陣列格式：`[id, 時間, 亮度, 溫度]`，依時間最新到最舊排序)*

---

### 3. 匯出下載 CSV (Export to CSV)
- **路徑**：`/iot_csv/{item_count}`
- **方法**：`GET`
- **參數說明**：
  - `item_count` (必填)：欲導出下載的最新資料筆數。

#### 👉 測試範例
- **請求連結**：直接在瀏覽器輸入或使用下載工具請求：
  ```
  http://127.0.0.1:8000/iot_csv/10
  ```
- **預期行為**：瀏覽器會自動下載名為 `downloaded_file.csv` 的檔案，內容如下：
  ```csv
  時間,亮度,溫度
  20260706 12:35:10,120.5,26.4
  20260706 10:00:00,98.0,25.5
  ```
